# 前言

## 本书解决什么问题

许多团队已经能让 Agent 调用工具，却很难让 Agent 稳定、可复用、可治理地完成复杂任务。问题通常不在模型不会推理，而在能力没有被工程化封装：流程散落在 prompt 里，触发条件模糊，输入输出不可测试，失败无法回滚，版本升级没有回归。

本书把 Skill 定义为：面向特定任务的可执行流程契约。它是一种程序性知识封装，告诉 Agent 什么时候做、怎么做、做到什么标准、出错怎么办。

## 读者

- AI 应用工程师：希望把经验流程沉淀为可复用能力。
- Agent 开发者：希望设计更稳定的 tool use 与 workflow。
- 平台工程师：希望建设内部 Skill 仓库、权限、测试与发布体系。
- 技术负责人：希望评估 Skill 体系是否具备生产可用性。

```mermaid
  K --> D
```

## 写作规则

每章都按工程手册写法组织：先讲问题，再给方法，然后给模板、反例、练习和检查清单。项目和论文不会写成综述，而会转译成可执行的 Skill 编写原则。

## 成书标准

全书至少包含 4 个完整 `SKILL.md` 实战案例、10 个行业案例卡片、8 类可复制模板、5 张核心图，以及可执行风格的回归测试样例。
