# 感知层设计审查报告

> 审查日期：2026-05-31
> 审查范围：ball-perception 感知层全模块（Detector / Tracker / Filter / Geometry）

---

## 一、整体架构评价

感知层的四模块流水线设计 **架构合理**：

```
Detector → Tracker → Filter → Geometry → 3D State
(检测)     (跟踪)    (滤波)    (重建)
```

这个管线是球类感知的经典流程，与 DeepMind 乒乓球机器人、ETH 羽毛球机器人的感知架构一致。模块间通过 `DetectionResult → TrackPoint → BallState → 3D Point` 的数据流串联，接口清晰。

## 三、核心问题：单帧球体检测（ball-detector）

**YOLO 模型确实没有真正部署和测试。** 具体情况如下：

### 3.1 YOLO 的现状：代码骨架存在，但无法运行

`detector.py` 中 `BallDetector` 类的 `load_model()` 方法：

```python
def load_model(self):
    from ultralytics import YOLO
    self.model = YOLO(self.model_path)  # 需要 yolov8n.pt 文件
```

**问题**：
- `ultralytics` 库**未安装**（当前环境只有 torch，没有 ultralytics 和 opencv-python）
- `yolov8n.pt` 模型文件**不存在**于项目中
- 即使安装了 ultralytics，`yolov8n.pt` 是 COCO 预训练模型，**不是球类专用模型**——它检测 80 类通用目标，虽然包含 "sports ball" 类，但对乒乓球/羽毛球这种小目标的检测精度极差

### 3.2 测试的"障眼法"

检测器测试中：

```python

# HSV 测试——被 pytest.importorskip("cv2") 跳过了
def test_hsv_detect_with_color(self):
    cv2 = pytest.importorskip("cv2", reason="opencv-python not installed")
    # ... 永远执行不到
```

**测试结果**：23 passed, **5 skipped**——被跳过的恰恰是所有涉及实际检测功能的测试（HSV 检测 2 个 + Pipeline 3 个），而通过的 YOLO 测试只是检查了构造函数参数赋值。

### 3.3 当前测试实际验证了什么？

| 通过的测试 | 实际验证内容 |
|-----------|------------|
| `test_yolo_init` | 参数赋值正确 ✅ |
| `test_detection_result_fields` | dataclass 字段正确 ✅ |
| Filter 10 个测试 | Kalman 滤波数学正确 ✅ |
| Geometry 6 个测试 | 三角测量数学正确 ✅ |
| Tracker 5 个测试 | 滑动窗口平滑正确 ✅ |

**结论**：下游三个模块（Tracker/Filter/Geometry）的数学逻辑是正确的，但**整个感知管线的入口——检测器——从未被真正验证过**。这就像验证了一条高速公路的路面质量，但入口收费站还没建好。

## 四、其他设计问题

### 4.1 YOLO 检测球类的根本性问题

通用 `yolov8n.pt` 对球类检测存在先天不足：
- **乒乓球**：直径 40mm，在 1280×1024 图像中仅占 ~10×10 像素，YOLO 的下采样会丢失目标
- **羽毛球**：高速运动（可达 400 km/h），运动模糊严重，单帧检测极难
- **网球**：相对可行，但需要 fine-tune

### 4.2 ONNXBallDetector 的后处理有 bug

`detector.py` 中 ONNX 后处理假设输出格式为 `[x1, y1, x2, y2, conf, ...]`，但 YOLOv8 导出的 ONNX 输出格式是 `[cx, cy, w, h, conf, ...]`，坐标解码逻辑不匹配。

### 4.3 Pipeline 默认配置用 YOLO 但测试用 HSV

`config.yaml` 默认 `detector.type: "yolo"`，但回归测试中 `TestPipeline` 的 `make_dummy_config()` 用的是 `type: "hsv"`——配置和测试不一致。
