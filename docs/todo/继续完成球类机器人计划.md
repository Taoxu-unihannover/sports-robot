## 修复总结

### 问题根因

`SuccessRateCallback` 和 `CurriculumCallback` 对 success_rate 的统计差异来自**统计粒度不同**：

| Callback | 统计方式 | 结果 |
|---|---|---|
| **SuccessRateCallback**（修复前） | 遍历**每一步**的 info，包括非终止步 | ≈0%（因为非终止步 success=False 占绝大多数） |
| **CurriculumCallback** | 只统计 **done=True** 的终止步 | 40.3%（正确的 episode 级成功率） |

### 根本原因

环境 [summit_catcher_v1.py](file:///home/xutao/文档/sports-robot/dynamic-tennis-v2/scripts/envs/summit_catcher/summit_catcher_v1.py#L900-L902) 在**每一步**都返回 `info["success"] = bool(caught)`：
- 非终止步：`caught=False` → `success=False`
- 终止步：`caught=True/False` → `success=True/False`

一个 800 步的 episode 中，只有最后 1 步可能 `success=True`，其余 799 步都是 `False`。所以按步统计的成功率 ≈ 1/800 ≈ 0.125%，几乎为 0。

### 修复方案

将 `SuccessRateCallback` 改为和 `CurriculumCallback` 一样，**只统计 done=True 的终止步**：

```python
# 修复前：遍历所有步
for info in infos:
    if info.get("success") is not None:
        self.buffer.append(float(val))

# 修复后：只统计终止步
dones = self.locals.get("dones", None)
if dones is not None:
    for i, done in enumerate(dones):
        if not done:
            continue
        info = infos[i] if i < len(infos) else {}
        success = info.get("success", info.get("is_success"))
        if success is not None:
            self.buffer.append(float(success))
```
