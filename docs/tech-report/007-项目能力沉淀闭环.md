# 005 — 项目能力沉淀闭环

> 版本：1.0 | 日期：2026-06-09
> 范围：从已有项目抽取能力 → 沉淀为 skills/plugins → 自举新项目 → 性能验证 → 能力补齐

## 1. 闭环流程

```
┌─────────────────────────────────────────────────────┐
│                    闭环流程                          │
│                                                     │
│  ① 冻结 baseline ──→ ② 选择 skills/plugins         │
│        ↑                          ↓                 │
│  ⑦ 性能判定 ←── ⑥ 对比 baseline ←── ③ 生成项目    │
│        │                          ↑                 │
│        ↓                          │                 │
│  ⑧ 更新 skills ←── ⑤ 归因失败 ←── ④ 运行验证     │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 1.1 步骤详解

| 步骤 | 输入 | 输出 | 关键约束 |
|---|---|---|---|
| ① 冻结 baseline | 源项目 | baseline 指标 + 可复现命令 | 指标必须可量化 |
| ② 选择 skills | 项目需求 + skills 清单 | 使用的 skills 列表 | 不得直接复制源码 |
| ③ 生成项目 | skills + plugins | 新项目代码和配置 | 只使用 skills 生成 |
| ④ 运行验证 | 新项目 | smoke test + 训练 + 评估 | 必须通过 check_env |
| ⑤ 归因失败 | 验证结果 | 失败类型 → skill 归因 | 归因到 skill 层 |
| ⑥ 对比 baseline | 新项目指标 + baseline | 差距报告 | 同配置同种子 |
| ⑦ 性能判定 | 差距报告 | PASS / PASS_WITH_GAP / FAIL | 按验收标准判定 |
| ⑧ 更新 skills | 归因结果 | 更新的 SKILL.md / scripts | 只抽象模式，不复制代码 |

## 2. 失败归因矩阵

| 失败类型 | 归因 skill | 修复方向 |
|---|---|---|
| 环境无法加载 | mujoco-tennis-world-builder / gymnasium-mujoco-env-builder | 重设计资产路径、XML 模板、环境初始化 |
| 训练脚本崩溃 | sb3-rl-training-runner | 重设计训练配置、callback、模型保存加载 |
| reward 不收敛 | gymnasium-mujoco-env-builder / truth-state-policy-input | 重设计任务、观测、奖励、终止条件 |
| 感知误差过大 | sim-camera-perception-input / ball-perception | 重设计相机、渲染、检测、状态估计 |
| 评估不可比较 | mujoco-policy-evaluator | 重设计指标、随机种子、baseline 固化 |
| 可视化缺失 | robot-trajectory-web-visualizer | 重设计 episode schema 与导出器 |
