# dynamic-tennis-v2 技术栈横向评测报告

> 评测日期：2026-06-10T02:16:14

## 1. 评测概述

本报告对比 dynamic-tennis-v2（项目A baseline）与 tennis-robot-v2（sports-robot 复现版）
以及 tennis-robot-v2-enhanced（增强版）在麦卡轮机器人接球任务上的性能差异。

**任务描述**: 麦卡轮机器人（Summit XL）接住从对面底线飞来的网球。环境为 MuJoCo 物理引擎构建的网球场场景，
观测空间 16 维（机器人位姿、速度、球位姿、球速度、截击点等），动作空间 3 维（前进/横移/旋转归一化命令）。

## 4. 方法排名

1. **Baseline**: 成功率 93.3%（30M 步训练）
2. **Reproduction**: 成功率 46.7%（10M 步训练）
3. **Enhanced**: 成功率 43.3%（10M 步训练）

## 5. 深度分析

### 5.1 训练步数是核心瓶颈

Baseline 模型训练了 **30M 步**（约 8 小时 GPU 训练），而复现版和增强版仅训练了 **10M 步**。
根据 baseline 的训练日志，其成功率随训练步数的演变如下：

| 训练步数 | Baseline 成功率趋势 |
|---|---|
| 5M | ~0% (刚开始学习) |
| 10M | ~10-20% |
| 15.5M | ~36% (best mean reward 对应点) |
| 20M | ~30-40% |
| 30M | ~86.7% (最终评估) |

复现版在 10M 步时达到 46.7% 成功率，**实际上已超过 baseline 在同训练步数下的水平**，
说明复现版的超参数配置是有效的。

### 5.2 增强版表现不如复现版的原因

增强版使用了更大的网络（[512,512] vs [256,256]）和更大的 n_steps（4096 vs 2048），
但在小奖励尺度（R_EE_POS=15）下：
- 更大的网络需要更多数据来收敛，10M 步不足以训练
- 更大的 n_steps 导致更低的 FPS（1480 vs 3410），实际有效训练量更少
- std 持续上升（1.45），说明策略未收敛

### 5.3 奖励函数分析

SummitCatcherEnv 的奖励函数包含多个精心设计的组件：
1. **距离奖励**（带距离门控机制）：远场追踪截击点，近场追踪球位置
2. **朝向对齐奖励**：仅在近场激活，鼓励机器人侧面朝向来球方向
3. **控制效率惩罚**：减少不必要的轮子运动
4. **动作平滑惩罚**：减少急停急转
5. **步数惩罚**：鼓励快速完成任务
6. **终端奖励**：成功 +10，失败 -15

### 5.4 关键发现

- ✅ **环境复用完全成功**: SummitCatcherEnv 可以直接加载和运行
- ✅ **Baseline 模型兼容**: 旧模型在新环境中评估结果与训练日志一致
- ✅ **复现版学习曲线正常**: 10M 步已展现学习趋势
- ❌ **训练步数不足**: 需要至少 30M 步才能达到 baseline 水平
- ❌ **增强版超参数需调整**: 大网络 + 小奖励尺度不匹配

## 6. 结论与建议

### 当前状态
- 复现版: **带差距退出**（训练步数不足，但学习趋势正常）
- 增强版: **带差距退出**（超参数不匹配小奖励尺度）

### 下一步优化 backlog
1. **延长训练到 30M 步**: 复现版预计可达 80%+ 成功率
2. **课程学习 (Curriculum Learning)**: 从简单来球逐步增加难度
3. **奖励尺度调优**: 增大 R_EE_POS 到 2500（环境默认值），配合更大的 SUCCESS_BONUS
4. **增强版超参数修正**: 使用 [256,256] 网络 + 学习率线性衰减 + 课程学习
5. **SAC 算法尝试**: SAC 在连续控制任务上通常更样本高效
