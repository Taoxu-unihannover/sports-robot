# GTSAM/ISAM2 异步因子图融合 Recipe

## 思路

球类机器人通常配备多个不同频率、不同延迟的传感器（双目相机 60 Hz、IMU 200 Hz、编码器 1 kHz）。传统 EKF 要求所有观测同步到同一时刻，丢帧或延迟时只能等待。GTSAM/ISAM2 将状态估计建模为因子图，每个观测作为独立因子加入，增量求解器 ISAM2 只更新受影响的贝叶斯树分支，实现微秒级增量更新。

## 核心原理

### 因子图表示

将状态估计问题建模为因子图：

$$\max_x \prod_i \phi_i(x_i)$$

其中 $\phi_i$ 是因子（对应观测），$x_i$ 是该因子连接的状态变量。不同传感器的观测作为独立因子加入即可，无需等待同步。

### ISAM2 增量求解

当新观测到达时，ISAM2 只更新受影响的贝叶斯树分支：

- 全批优化：$O(n^3)$，每帧重算
- ISAM2 增量：近似 $O(1)$（稳态下），只更新变化部分

### 多模型支持

GTSAM 内建流形类型：

| 类型 | 用途 | 球类机器人场景 |
|---|---|---|
| $\mathrm{SO}(3)$ | 旋转 | 拍面姿态、IMU 姿态 |
| $\mathrm{SE}(3)$ | 刚体变换 | 机器人末端位姿 |
| $\mathrm{S}_2$ | 单位球面 | 球方向观测 |
| IMU 预积分 | IMU 因子 | 相机帧间 IMU 积分 |

### 在线外参标定

将相机外参 $T_{CI}$ 作为状态变量加入因子图，随运行持续精化。长时间运行后，利用回环因子消除累积漂移。

## 步骤

1. **构建因子图**：定义状态变量（球位置、速度、旋转、机器人构型）和观测因子（相机投影、IMU 预积分、编码器读数）。
2. **异步因子添加**：每个传感器观测到达时，按自身时间戳创建对应因子，插入因子图。
3. **ISAM2 增量更新**：调用 `ISAM2.update()` 增量求解，获取最新状态估计和协方差。
4. **外参精化**：将相机-IMU 外参作为可优化变量，随运行持续精化。
5. **协方差提取**：从 ISAM2 的贝叶斯树中提取边际协方差，传递给风险评估模块。

## 关键参数

| 参数 | 建议值 | 说明 |
|---|---|---|
| ISAM2 重新线性化阈值 | 0.1 | 控制增量更新的精度 |
| 因子图滑动窗口 | 50–200 帧 | 平衡精度与内存 |
| IMU 预积分间隔 | 与相机帧对齐 | 两帧之间积分 |
| 外参先验噪声 | 1 mm / 0.1° | 初始标定精度 |

## 验收指标

| 指标 | 建议 |
|---|---|
| 增量更新耗时 | < 0.5 ms（稳态） |
| 状态估计精度 | vs EKF 提升 > 20% |
| 异步丢帧容忍 | 丢帧 30% 时精度退化 < 10% |
| 外参精化收敛 | 运行 10 min 后外参变化 < 0.1 mm |
| 协方差一致性 | NEES 在 95% 置信区间内 |
