# DeepMind 延迟分布建模 Recipe

## 思路

DeepMind 在 Nature 2026 论文中指出：感知延迟不是固定值，而是一个概率分布。将延迟分解为曝光、传输、推理、估计、通信等环节，每个环节独立建模均值和方差。在 EKF 中，延迟方差膨胀观测噪声协方差 $R_k$，使滤波器在延迟抖动大时自动增大不确定度，控制层据此选择更保守的策略。

## 核心原理

### 延迟分解与建模

总延迟 $\delta_k = \sum_i \delta_k^{(i)}$，各环节独立建模：

| 环节 | 典型均值 | 典型方差 | 建模方式 |
|---|---|---|---|
| 相机曝光 | 1 ms | 0.1 ms² | 全局快门近似常数；滚动快门按行建模 |
| 图像传输 | 3 ms | 0.5 ms² | USB/GigE 传输抖动 |
| 检测推理 | 5 ms | 1 ms² | GPU 推理时间波动 |
| 状态估计 | 2 ms | 0.2 ms² | EKF 计算时间 |
| 通信延迟 | 1 ms | 0.3 ms² | 网络抖动 |

总延迟均值 $\bar\delta = \sum_i \bar\delta^{(i)}$，总方差 $\sigma_\delta^2 = \sum_i \sigma_i^2$。

### 协方差膨胀

延迟不确定性导致观测对应的状态时刻不确定，等效为额外的位置噪声：

$$R_k' = R_k + \underbrace{F_k \cdot \sigma_\delta^2 \cdot \hat v_k \hat v_k^\top}_{\text{延迟膨胀项}}$$

其中 $\hat v_k$ 是当前球速度估计，$F_k$ 是状态转移雅可比。球速越快、延迟方差越大，膨胀项越大。

### 分层策略与延迟适配

| 延迟条件 | 策略层 | 模型复杂度 | 典型动作 |
|---|---|---|---|
| $\sigma_\delta < 2$ ms | 层3（精确击球） | 完整动力学+接触 | 扣杀/精准回球 |
| $2 \le \sigma_\delta < 5$ ms | 层2（定位） | 简化动力学 | 推挡/防守 |
| $\sigma_\delta \ge 5$ ms | 层1（准备） | 几何模型 | 移动到预测区域 |

## 步骤

1. **离线统计延迟分布**：在部署前，用硬触发/闪光同步采集大量感知延迟样本，统计各环节均值和方差。
2. **在线延迟估计**：运行时，通过时间戳差值持续更新延迟分布参数（可用 RLS 慢环更新）。
3. **EKF 协方差膨胀**：在每次观测更新时，根据当前延迟方差膨胀 $R_k$。
4. **风险门控**：将膨胀后的协方差传递给风险评估模块，计算命中概率和 CVaR。
5. **策略降级**：当延迟方差超过阈值时，自动降级到更保守的击球策略。

## 关键参数

| 参数 | 建议值 | 说明 |
|---|---|---|
| 延迟统计窗口 | 100–500 帧 | 用于在线更新延迟分布 |
| 膨胀系数 | 1.0–2.0 | 调节膨胀项的保守程度 |
| 策略降级阈值 | 2 ms / 5 ms | 中/高风险延迟方差阈值 |
| 延迟异常检测 | $>3\sigma$ | 触发重新标定警告 |

## 验收指标

| 指标 | 建议 |
|---|---|
| 延迟估计精度 | 均值误差 < 1 ms |
| 协方差一致性 | NEES 在 95% 置信区间内 |
| 延迟抖动下命中率退化 | < 10%（vs 固定延迟补偿） |
| 策略降级触发率 | < 5%（正常工况下） |
